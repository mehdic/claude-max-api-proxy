# claude-proxy v1.0.5

Fortress hardening release for the local OpenAI-compatible Claude CLI proxy.

## Highlights

- Added Claude CLI capability detection before passing optional/version-sensitive flags.
- Added live SDK/client matrix covering fetch, Python stdlib, optional OpenAI Node, optional OpenAI Python, and optional LangChain JS.
- Added deeper live failure simulation for invalid requests, client aborts, trace headers, and tool-bridge resilience.
- Added optional durable SQLite trace backend with redacted completed trace records.
- Hardened stream-json canary behavior so it validates transport and bounded shutdown rather than brittle model obedience.
- Kept OpenClaw authoritative for caller-dispatched external OpenAI/OpenClaw tool execution.

## Validation

- `npm test -- --runInBand --silent` → 196 passing
- `npm run canary:stream-json` → haiku, sonnet, opus passed

## Notes

The SDK matrix skips optional SDK packages unless installed. Tool-call checks may skip while the local Claude subscription is returning limit/reset text instead of producing tool-call content.
